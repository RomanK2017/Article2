The instant image matching algorithm

Roman Kovalenko

 

Matching of image objects is a very common application of computer vision technologies. The combination of machine learning and image segmentation methods could solve this problem very well. However, sometimes it is impossible to use machine learning techniques and the only way is using a pixel by pixel comparison of image blobs. An accurate instant matching, which is a scale and rotation invariant, and is also insensitive to noise, requires special algorithms. The present paper is introducing a solution of the problem by the instant matching algorithm (IMA).

 

INTRODUCTION

          The purpose of the current research is to develop a simple and efficient Python based tool which will allow a comparison of two images (contours) with one line of code and with standard dependencies like Scikit and OpenCV.

A lot of matching algorithms already exist like corners, edges and curvatures detectors. A variety of approaches to estimate a shape’s similarities are already developed, hence, their application in software is sometimes complicated. The reviewing and comparison of the algorithms is not performed in this article.

          The proposed algorithm has to be a function which can conduct a scale, rotation invariant, and noise insensitive comparison of a couple of images (contours) and return Y/N result. Code optimization is postponed and will be done at a later stage of the research.

 

THE PROPOSED APPROACH

In the situation where it is not possible to use the power of machine learning algorithms, we still have some tools and approaches that can be applied. This area is on the cutting edge of modern computer vision technology - a lot of efficient algorithms and programming tools have been developed, addressed to a particular problem.

As assumed, the proposed algorithm deals with contour comparison in the grey scale image format, it means that filtering, smoothing, and object segmentation stages are not a topic of the current research. It is supposed that correctly extracted image contours might be matched to a template contour with a certain accuracy and insensitivity to noise, at the same time the matching has to be a rotation and scale invariant.

This approach could be very valuable if there is no prior knowledge about a template object and only a single example of the template is available. For example, there is an image with an assorted pile of objects, and a template image. We have to compare each object candidate from the original image with the template and count matches. The contour extraction process and image quality may cause some noise. This is the example where the application of machine learning technology is very limited but the proposed algorithm could be an efficient and simple solution.

The algorithm is based on a combination of statistical methods and contour shape descriptors. The first stage is getting a template and a candidate contour image (image1 and image2). It is assumed that only one contour object is on a candidate image and all irrelevant small contours and random objects, caused by noise, must be excluded on the segmentation phase. If there are many contours, they have to be compared separately one by one.     

 

IMAGE PREPROCESSING

 

The pre-processing phase is applied to each image and includes a few stages.

Stage1

Both input images, image1 (h1,w1) and image2 (h2,w2),  have to be resized to the size (max(h1,h2),max(w1,w2)).

 

                                                                   (1)

 

Stage2

Transform each image into grey scale and apply MIN_MAX normalization to pixels, their brightness has to be in [0,255] range.

Stage3

The average brightness Bave (only pixels inside the contour) is calculated, and applied as a threshold. All pixels darker than Bave are replaced by 0 (black), and other 255 (white).

Stage4

Get mask images of the filled external contours from the images 1 and 2. Find areas (A1 and A2) of the filled external contours.

Stage5

Get parameters of the rotated enclosing rectangle of the external contours. The parameters are 4 points with (x,y) coordinates which allow to identify a contour’s longest and shortest dimensions (rwidth and rheigth), and the angle of rotation. The example is shown on Picture 1. This stage is applied to both images. Width and height are rotation invariant parameters.

Picture 1 – The example of an object (white spiral), external contour of the object (red), and enclosing rotated rectangle (blue)

 

For some objects like a circle or a square it is impossible to identify width and height and angle of rotation. Circular objects have no enclosing rectangle with uniquely defined coordinates. Therefore, this stage might be irrelevant and skipped in some cases.

Stage6

Identify the centroid’s coordinates (Cx,Cy) and radius R of the minimal enclosing circle of the filled external contour. It has to be performed for images 1 and 2. Then the list of distances d of all white pixels to the centroid must be created, where the distance to i-pixel (formula 2). The distance d is not a Euclidian distance, this is a scale invariant value. The area of an external contour A under the square root transforms the value into unitless form.

 

                                                                                                    (2)

 

Stage7

Call function Get_KS_Pi with parameters (mask image of a filled external contour; a tuple of coordinates of an enclosing rectangle; an area of an external contour). This function creates 4 lists d1, d2, d3, and d4, the distances of all white pixels from each corners of the enclosing rectangle. Then the lists are organized into pairs , because there are 4 points, 6 unique pairs can be generated : (d1,d2), (d1,d3), (d1,d4), (d2,d3), (d2,d4), (d3,d4). Then Kolmogorov-Smirnov test is applied to each pair and their statistics (not p-value) are added in KS_Pi list. KS_Pi is a 6-component reversed sorted vector. Two vectors have to be generated (image1 and image2).

Stage8

Identify internal contours and their parameters. An input image and a template may (or may not) have internal contours. Actually, internal contours are patterns of empty areas (black pixels) inside an external contour and are separated by a white border from any other contours (Picture 2).

Picture 2 – Internal contours (green) of the object with an external contour (red)

All parameters must be scale invariant and unitless. The first parameter is Internal_cont_r_areas, this is a sorted reversed list of a unitless ratios of areas (Formula 3).

 

                                                         (3)

 

Ai – areas of an i-internal contour, n – number of internal contours, A – area of an external contour.

There are two requirements to the components of Internal_cont_r_areas:

-        Exclude all components <1% value (it is assumed they are insignificant or might be a result of resizing or noise)

-        If the sum of all components of Internal_cont_r_areas is less than a certain threshold (mentioned below), then it is supposed that all internal contours are not large enough for a comparison of their similitude.

The second parameter of internal contour’s description is Internal_cont_r_centres. This is a list of centroids of all relevant internal contours.

Stage9

At this stage the list Internal_c_DCREC is generated. The components of Internal_cont_r_areas, are unitless descriptors of location of internal contours. Each descriptor is a ratio of distance between a couple of contour’s centroids and radius of an enclosing circle of an external centroid (Formuls3). The list must be generated for each image and to be reversely sorted.

 , j>i                                                      (4)

 

RE – radius of the enclosing circle of an external contour. i,j – numbers of internal contours which are used for generating unique pairs of centroids [Ci,Cj] , j>i, where Ci is Ci(Cx,Cy) – coordinates of the centroid of i - internal contour. This is the last stage of calculating the input parameters

Stage10

Image pre-processing (Stages 1-4) and calculation of the input parameters (Stages 5-9) are completed and the instant matching algorithm (IMA) can be used. There are 4 possible conditions of a couple of input images, they are called Cases 1-4 of IMA (Picture 3).

 

Picture 3 – The structure of the instant matching algorithm (IMA)

 

 

THE INSTANT MATCHING ALGORITHM

 

The algorithm works with two images, each image contains only one object (contour). A contour might (or might not) contain internal contours. If an input image has many objects, they must be extracted separately and saved as separate images of mask arrays. It requires the application of filtering and image segmentation techniques. However, these techniques are not a part of the current research and will not be discussed further.

 

CASE1

How different are rwidth/rheigth ratios of two compared external contours? If the ratios are significantly different (exceeding T1 threshold), it is assumed that contours are different (formula 5).

 

                                                                                               (5)

 

rw,rh – width and height of an external contour, derived from coordinates of rotated enclosing rectangle, threshold T1=10%.

If the ratio’s difference exceeds the threshold, it means the objects are mismatched, it is the end of the procedure with N result. This is very simple and fast test that could save a lot of computational time. If the difference is not significant, the tests of Case 2 have to be used.

 

CASE2

Is it possible to identify height and width of a contour? If the contour is a square, the height is equal to width, if the contour has circular symmetry, it is not even possible to uniquely define coordinates on the enclosing rotated rectangle. Case 2 tests possible circularity of the external contours. If a contour is possibly circular, then (rwidth/rheight)=1. This is an ideal situation, for the practical application the threshold T2=5% was applied. At least, one of two external contours have to satisfy the condition (formula 6) for accepting that they are not circular.

 

                                                            (6)

 

If contours are not circular, the points of enclosing rectangle can be identified. The statistics (not p value) of Kolmogorov-Smirnov test, applied to the 6 components vectors KS_Pi1 and KS_Pi2, saved as P4_test. KS_d1_d2 keeps p-value of Kolmogorov-Smirnov test, applied to d1 and d2. These are parameters, calculated at stages 6 and 7.

The empirically obtained thresholds are: T3=0.075 and T4=1E-90.

If (P4_test>= T3) AND (KS_d1_d2 >= T4) then assume that contours are matched, otherwise, they are not matched.

If contours did not pass the aforementioned circularity test (formula 6) then it is assumed, they are, possibly, circular and the approach of Case2 cannot be applied, this condition requires Case3.

 

CASE3

Both contours have been assumed as possibly circular. Case3 supposes similarity of internal contour’s locations and areas. However, the scale invariance requires the internal contour’s parameters to be unitless. Criterion3 is the similarity test of internal contours, returned by function Criterion3_test. Criterion3_test requires parameters (A1,A2 – areas of external contour; Internal_cont_r_areas (1 and 2); Internal_c_DCREC (1 and 2)) from stages 4,8, and 9. Also, function Criterion3_test needs 3 thresholds: T5=t1=15%; T6=t2=15%; T7=t_min_area=7%;

 

Function Criterion3_test:

Areas = Internal_cont_r_areas (1 and 2),
Dist = Internal_c_DCREC (1 and 2),
Aext = A (1 and 2).

If total sums of Areas of both contours are less than the threshold T7 =7% (formula 7), then it is accepted, that objects do not have any internal contours, thus, they are both similar in the sense of Criterion3, and the function returns Criterion3=Y.

 

                                                            (7)

 

Noise and image transformation such as rotating, and resizing often causes a subtle distortion of the original image which can appear as new small (sometimes just a size of a few pixels) internal contours. The low value of T7 can lead to an increase of false negative errors. 

If the total sum of contours exceeds T7, only relevant n contours should be chosen for further analysis, n=min (n1, n2), where n1 and n2 – original number of detected internal contours (images 1 and 2).

The similarity of areas of internal contours requires a comparison of the list of differences LOD (formula 8) with the threshold T5 (formula 9). The sum of all elements of LOD equal to 0 if internal contour’s areas are identical. 

 

                                         (8)

                                                                                                       (9)

 

If the condition (formula 9) is true for all LOD’s components, then the similarity of the internal contour’s areas is confirmed (C3_areas=Y). The next step is the similarity test of location of the internal contours. This test is pretty similar to the aforementioned area’s test, applied to LODist (formula 10), LODist is the list of differences of distances between a pair of centroids of the internal contour.

 

                                               (10)

 

Location similarity is confirmed (C3_location=Y) if all elements of LODist are below the threshold value T6 (formula 11).

 

                                                                                                             (11)

 

Three possible outcomes of the function Criterion3_test:

1.     No internal contours then Criterion3=Y

2.     If C3_areas=Y and C3_location=Y then Criterion3=Y

3.     Otherwise Criterion3=N

After obtaining Criterion3, equivalent diameter ED must be calculated (formula12). Equivalent diameter of a contour is a diameter of a circles which has the same area A as the contour.

 

                                                                                                          (12)

         

Equivalent diameters are as a part of the expression (formula 13) of dimensionless values DC, which is used for comparison of geometrical similarity of the external contours 1 and 2.

 

                                                                                         (13)

 

A – area of an external contour; ED – equivalent diameter; R – minimal radius of an enclosing circle of an external contour.

The value of geometrical similarity - DC_test has to be found (formula 14).

 

                                                                                  (14)

 

Function Matching_Criteria is the final step of Case3. The function requires 5 input parameters:

DC_test – geometrical similarity
KS_d1_d2 - p-value of Kolmogorov-Smirnov test, applied to d1 and d2
Criterion3 – similarity of the internal contours
rc - minimal size ration of the input images (formula 15)
noise_level – level of “salt and pepper” noise (0-1), shows the ratio of noise pixels to all pixels of the image, default value is 0. IMA is adjusted to the range 0-0.25.

 

                                                                                      (15)

 

The function compares input parameters with the thresholds T8 and T8 (formulas 16 and 17)

 

                                                                                 (16)

                                                                           (17)

 

The function returns ‘Y’ if all values satisfy the condition (formula 18), otherwise ‘N’

 

                               (18)

 

If the function returns ‘N’, it means that contours are not matched but if the return is ‘Y’ the test of CASE4 must be passed.

 

CASE4

Sometimes potentially circular contours might be recognized as matched by CASE3 test. They have similar DC_test, KS_d1_d2, and Criterion3 but they are different (example – Picture 4).

Picture 4 – Images incorrectly matched by CASE3 test

 

It was expected that such samples would have similar DC_test, KS_d1_d2 but Criterion3 has to detect the difference of the structure of the internal contours. What if they do not have internal contours? It is the exact problem of the images on Picture4.

          The solution is an update of the original images by enclosing them into circles (Picture5). As the result, new internal contours might be created in the space between an outer border of an external contour and an internal border of an enclosing circle. After this, Criterion3_test function returns a new Criterion3 result, which is the final conclusion about contour’s similitude.

          The centre of an enclosing circle is a corresponding centroid. The calculated radius Ri and thickness of the inclosing circle have to be adjusted (formulas 19 and 20).

                                                                                                          (19)

                                                                                               (20)

          The areas of the updated external contours have to be recalculated as well (Stage 4). Other parameters of the new internal contours can be found by passing Stages 8 and 9 again.

 

Picture 5 – CASE4, enclosing circle

 

          The second Criterion3_test must be performed with a new set of the thresholds (T5=5%, T6=5%, T7=5%). Now the similarity of the original contours is confirmed, if Criterion3=Y.

PARAMETERS OF IMA

 

This is always a trade-off to make a model sensitive enough to detect a subtle difference between a couple of graphical objects but, at the same time, make it insensitive to noise and distortions. It is the cause why known methods, such as Kolmogorov-Smirnov test, applied with new parameters and a new sense behind them.

          Non-parametric Kolmogorov-Smirnov test is a very efficient method to compare a couple of distributions and estimate p-level base on the maximal distance between two cumulative curves. The algorithm has the same purpose, namely, comparing distribution d1 and d2 (KS_d1_d2) or comparing distances between pixels and an enclosing rectangle (P4_test) but the difference is the choice of the level of acceptance of hypothesis about similarity of distribution.

          The level of acceptance is dependent on image properties, noise, resolution, required sensitivity etc., and must be adjustable to a specific practical requirement. The thresholds T3,T4, and T8,  of the algorithm, related to KS test, are entirely empirical.

          Other parameters of IMA (DC_test and Criterion3) describe geometrical similarity of contours with empirically identified thresholds (Table 5.) Particular tasks may require an adjustment of the default thresholds, the recommendations are not developed yet.

DISCUSSION

 

The parameters of IMA might require an update for better performance if all objects are circular, or if all objects are very similar and a high sensitivity is required. Also, it was not explored how noise affects internal contour’s structure and, thus, how T4-T7 thresholds should be adjusted. Potentially the application of IMA could be more versatile but it requires a lot of experiments and developing of recommendations. The current paper is only an introduction with the set of the default parameters which allow to minimize the total number of errors, and make an unbiased classifier with regards to FN and FP.

          Another area of the following research is an optimization of the code and reduction of the processing time and memory usage. The present paper does not touch on this topic at all.

          As it was mentioned above, some objects with complicated internal contour’s structures and very thin borders between them (example SP4) could be very sensitive to an image transformation. The problem could be solved by recovering the shape and borders of internal contours, making all lines and curves smooth and continuous. This is the topic of further research as well.

          The proposed algorithm was not compared to other existing technologies of computer vision. It was an attempt to transfer a machine learning and computer vision problem to computer vision problem only. In other words, if we cannot use the power of machine learning, how can we classify images with a mathematical approach and look at the problem as a computer vision problem only. This was the main focus of the present paper and it was successfully done.

import numpy as np

import cv2

import scipy.stats

from itertools import chain

import os

 

def Get_KS_Pi(ext_c_img,Pi_xy,A):

    '''

    P1,P2,P3,P4 - corners of the rotated enclosing rectangle

    calculate arrays of distances from each Pi to each point of a filled outer contour

    get a sorted array of pairs (Pi,Pj) and KS test

    di -list of distances from cornets to all not black pixels of mask(filled outer contour)

    pairs - list of unique combinations of corners, 6 pairs

    KS_Pi- sortet list of KS DISTANCES(not p)!! of pairs of di, 6 numbers

    It is supposed that similar not circular contours have similar KS_Pi

    '''

    di=[]

    tmp = np.where(ext_c_img != [0])

    for point in Pi_xy:

        Cx=point[0]

        Cy=point[1]

        di.append(np.sqrt( ((tmp[1]-Cx)**2+(tmp[0]-Cy)**2)/A))

    pairs=[]

    for i in range (0,len(Pi_xy)-1):

        for j in range (i+1,len(Pi_xy)):

            pairs.append([i,j])

    KS_Pi=[]

    for p in pairs:

         KS_Pi.append(scipy.stats.ks_2samp(di[p[0]], di[p[1]])[0])

    KS_Pi.sort(reverse = True)

    return KS_Pi

 

def Matching_Criteria(KS_test_p, DC_test, Criterion3,rc,noise_level=0.0):

# Matching images.

#rc - minimal size ratio of a couple of images of (h1/h2,h2/h1,w1/w2,w2/w1)                                                                      

    Matched_C='N'

    #

    KS_threshold=10**((-1.0)*(20+15*(1-rc)+0.05*noise_level))

    DC_threshold=5+0.1*noise_level+2*(1-rc)

    if (KS_test_p>=KS_threshold) and (DC_test<=DC_threshold) and (Criterion3=='Y'):

        Matched_C='Y'   

    return Matched_C

 

 

def Criterion3_test(Areas1,Dist1,Aext1,Areas2,Dist2,Aext2,t1=10,t2=10,t_min_area=5):

#Criterion3 test - similarity of internal contour's structure

#Areas1,Areas2 - desc. sorted lists of areas of internal contours, always <100%

# input areas arrays have contours >1% of area

#Dist1, Diar2 - desc. sorted lists of distances between internal contours 100%*[dist[i]/ER], always <100%

#Aext1,Aext2 - areas of outer contour; t1, t2 - criterion's thresholds

#t_min_area - minimal total sum on internal contour areas (% of ext contour) we assume that contours exist

#Lists MUST BE sortet!!!

    C3='N'

    C3_areas='N'

    C3_location='N'

    if np.sum(Areas1)<=t_min_area and np.sum(Areas2)<=t_min_area:

        # if contours in total less than 5% we assume it is noise/distortion. Neglect

        C3='Y'

        return C3

  

    n=min(len(Areas1),len(Areas2))

    LOD=[abs(Areas1[i]-Areas2[i]) for i in range(0,n)]

    # Check that all LOD elements are less than t1 and return areas test C3_areas='Y'

    if sum(map(lambda x : x>t1, LOD))==0:

        C3_areas='Y'

    #check that all contour locations ()distances are similar, test C3_location='Y'

    # look at n biggest contours if len(Dist1)<>len(Dist2)

    n=min(len(Dist1),len(Dist2))

    LOdist=[abs(Dist1[i]-Dist2[i]) for i in range(0,n)]

    if sum(map(lambda x : x>t2, LOdist))==0:

        C3_location='Y'

    if C3_areas=='Y' and C3_location=='Y':

        C3='Y'   

    return C3
